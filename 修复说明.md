# 登录问题修复说明

## 问题分析

在1Panel上构建Docker镜像后无法登录，主要有以下两个问题：

### 1. 数据库路径硬编码
- **问题**：代码中硬编码了Windows路径 `D:\桌面\code\school 2.1.1\go\school.db`
- **影响**：在Linux上无法连接到数据库
- **修复**：改为默认使用相对路径 `school.db`

### 2. Cookie设置问题
- **问题**：Cookie设置了 `HttpOnly=true` 标志
- **影响**：某些情况下JavaScript无法读取Cookie，导致登录状态无法保持
- **修复**：将HttpOnly标志设置为false

## 修改的文件

### 1. internal/repository/db.go
```go
// 修改前
dbPath = "D:\\桌面\\code\\school 2.1.1\\go\\school.db"

// 修改后
dbPath = "school.db"
```

### 2. internal/api/handlers/user.go
```go
// 修改前
c.SetCookie("session_id", sessionID, 86400, "/", "", false, true)

// 修改后
c.SetCookie("session_id", sessionID, 86400, "/", "", false, false)
```

### 3. scripts/init-db.go (新建)
- 数据库初始化工具
- 在容器启动时初始化数据库
- 创建默认管理员账户

### 4. Dockerfile (已更新)
- 添加了数据库初始化工具
- 添加了启动脚本
- 设置了GIN_MODE=release

## 使用方法

### 在1Panel上重新构建

1. **上传所有修改后的文件到Linux服务器**

2. **在1Panel中重新构建镜像**：
   ```bash
   # 删除旧镜像（可选）
   docker rmi school-app
   
   # 重新构建
   docker build -t school-app .
   ```

3. **重启容器**：
   ```bash
   # 停止旧容器
   docker stop school-app
   
   # 删除旧容器
   docker rm school-app
   
   # 启动新容器
   docker run -d -p 5000:5000 --name school-app school-app
   ```

4. **查看容器日志**：
   ```bash
   docker logs school-app
   ```

## 故障排除

### 仍然无法登录？

1. **检查容器日志**：
   ```bash
   docker logs school-app
   ```

2. **检查数据库文件**：
   ```bash
   docker exec school-app ls -la
   docker exec school-app ls -la school.db
   ```

3. **进入容器调试**：
   ```bash
   docker exec -it school-app /bin/sh
   ```

4. **检查数据库连接**：
   ```bash
   docker exec school-app ./init-db
   ```

### Cookie相关问题

如果仍然有Cookie问题，可以尝试：
1. 清除浏览器Cookie
2. 使用隐私/无痕模式
3. 检查浏览器控制台是否有错误

## 技术细节

### 数据库初始化流程

1. 容器启动时运行 `start.sh`
2. `start.sh` 先运行 `init-db` 工具
3. `init-db` 调用 `repository.InitDB()`
4. 自动创建用户表（如果不存在）
5. 自动创建默认管理员账户（如果不存在）
6. 启动主应用

### 会话管理

- 会话存储在内存中（map）
- 会话有效期：24小时
- Cookie有效期：24小时
- 使用随机会话ID
